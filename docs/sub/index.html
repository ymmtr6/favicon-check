パス別faviconのキャッシュ挙動

ブラウザは <link rel="icon"> をページ単位でキャッシュします。
Root → Sub と移動した際、タブのfaviconがすぐ切り替わるか確認してください。
切り替わらない場合、ブラウザがキャッシュした前のfaviconを使っています。
Chrome の挙動通常はページ遷移で即座に切り替わる。DevToolsのCache無効化で確実に確認可能
Safari の挙動apple-touch-icon は長くキャッシュされる（ホーム画面追加後は特に）
favicon.ico フォールバックルートの /favicon.ico が存在する場合、全パスで使われる場合がある
